How to Complete the Multi-Sig

  1. Generate the signing payload:
    node scripts/anchor-signature.cjs
    This creates signature-morpho-config.json containing the eip191Hash.
  2. ThirdWeb wallet signs: Open the ThirdWeb wallet at address 0xCD67f7e86A1397aBc33C473c58662BEB83b7a667 and call personal_sign on the eip191Hash from step 1. See documents/multisig-verification-walkthrough.md §2 for the full walkthrough.
  3. Coinbase wallet signs: Open the Coinbase wallet at address 0xDc2aFCd0a97c1e878FdD64497806E52Cc530f02a and sign the same hash. See documents/multisig-verification-walkthrough.md §3.
  4. Populate signatures in signature-morpho-config.json.
  5. Verify locally:
    node scripts/verify-multisig.cjs
  6. Submit the signed transaction to the network.
On-Chain Signature Verification: Once the contracts are deployed, the StoryAttestationService and StoryOrchestrationService contracts on Story Protocol (Chain 1514) validate EIP-191 signatures against the registered multi-sig signers on-chain. Use StoryScan or Basescan to verify each signature transaction after broadcast.

🌐 GitHub Pages Configuration

Recommendation: Use branch main with folder /docs as the GitHub Pages source. No build step is required — this docs/index.html is a static page and is served directly by GitHub Pages.

Steps to Enable GitHub Pages in Repository Settings

  1. Go to the repository on GitHub: slaps6331-cell/Millionaire-Resilience-LLC-JAH ↗
  2. Click Settings (top navigation bar).
  3. In the left sidebar, scroll to Pages.
  4. Under Build and deployment → Source, select Deploy from a branch.
  5. Under Branch, select main.
  6. Under Folder, select /docs.
  7. Click Save.
  8. Wait 1–2 minutes; GitHub will publish the site at:
    https://slaps6331-cell.github.io/Millionaire-Resilience-LLC-JAH/

Why main branch + /docs folder?

The repository uses a single main branch for all source code, contracts, and documentation. Keeping the frontend in /docs on main means:

  • No extra branch to maintain — every push to main automatically updates the site.
  • No build pipeline is needed for this static HTML page.
  • The deployment status, contract addresses, and multi-sig info on this page can be updated alongside the contract source in a single PR.

Relationship to Smart Contract Deployment

This GitHub Pages site serves as the public-facing documentation hub for the smart contracts. Once the contracts are deployed and real transaction hashes are available, update docs/index.html (or add a generated JSON file loaded by this page) to reflect live contract addresses — providing a single URL for auditors, investors, and counterparties to verify the on-chain deployment.

The site also documents the Morpho 2-of-2 multi-sig signer addresses, enabling independent on-chain verification of the 0xCD67f7e86A1397aBc33C473c58662BEB83b7a667 (ThirdWeb) and 0xDc2aFCd0a97c1e878FdD64497806E52Cc530f02a (Coinbase) wallets used for EIP-191 signature verification in the deployed contracts.

Automated Deployment via GitHub Actions (Optional)

A GitHub Actions workflow (.github/workflows/pages.yml) is included in this repository. It automatically deploys the docs/ folder to GitHub Pages whenever a commit is pushed to main. To use the Actions-based approach instead of the manual branch/folder setting above:

  1. In Settings → Pages → Source, choose GitHub Actions instead of Deploy from a branch.
  2. The workflow will trigger on the next push to main.

🚀 How to Deploy the Smart Contracts

Via GitHub Actions (Recommended)

  1. Set required GitHub Secrets (Settings → Secrets and variables → Actions):
    • DEPLOYER_PRIVATE_KEY — 0x-prefixed hex private key
    • ALCHEMY_API_KEY — Alchemy API key (Base L2 + Story Protocol RPC)
    • STORYSCAN_API_KEY — StoryScan API key (Story source verification)
    • ETHERSCAN_API_KEY — Basescan API key (Base source verification)
  2. Go to Actions → Deploy Smart Contracts.
  3. Click Run workflow and choose:
    • Network: story | base | both
    • Verify contracts: true
    • Dry run: false
  4. After success, download the deployment artifacts from the Actions run (deployment-config.<network>.json) and record the live contract addresses.

Via CLI (Local)

  1. cp .env.example .env and fill in all required keys.
  2. npx hardhat run scripts/deploy.cjs --network story
  3. npx hardhat run scripts/deploy.cjs --network base
  4. npx hardhat run scripts/post-deploy-orchestrate.cjs --network story
  5. npx hardhat run scripts/verify.cjs --network story
