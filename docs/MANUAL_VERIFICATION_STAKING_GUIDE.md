## 1. Pre-Verification Checklist

Before verifying any contract, confirm you have:

- [ ] Contract address (from deployment transaction receipt)
- [ ] Compiler version: `v0.8.26+commit.8a97fa7a`
- [ ] EVM version: `cancun`
- [ ] Optimization: `enabled` (200 runs; StoryAttestationService uses 1 run)
- [ ] Via IR: `true`
- [ ] License: `MIT`
- [ ] Constructor arguments: `0x` (all contracts use no-arg constructors)

### Compilation Settings (Must Match Exactly)

```
Compiler:     v0.8.26+commit.8a97fa7a
EVM Version:  cancun
Optimization: Enabled
Runs:         200 (except StoryAttestationService: 1)
Via IR:       true
License:      MIT
```

## 2. Manual StoryScan Verification (Chain 1514)

### Step 1: Navigate to StoryScan

1. Go to https://www.storyscan.io
2. Paste the deployed contract address in the search bar
3. Click the **Contract** tab
4. Click **Verify and Publish**

### Step 2: Select Verification Method

Choose **"Solidity (Standard-Json-Input)"** for the most reliable verification.

### Step 3: Enter Compiler Settings

| Field | Value |
|---|---|
| Compiler Type | Solidity (Standard-Json-Input) |
| Compiler Version | v0.8.26+commit.8a97fa7a |
| Open Source License | MIT License (MIT) |

Click **Continue**.

### Step 4: Upload Standard JSON Input

The Standard JSON Input is generated by Hardhat during compilation. To get it:

```bash
# In the repo root
cat artifacts/build-info/*.json | python3 -c "
import sys, json
data = json.load(sys.stdin)
print(json.dumps(data['input'], indent=2))
" > standard-json-input.json
```

Upload `standard-json-input.json` in the StoryScan form.

### Step 5: Enter Contract Details

| Field | Value |
|---|---|
| Contract Name | e.g., `contracts/StoryAttestationService.sol:StoryAttestationService` |
| Constructor Arguments | Leave empty (no-arg constructors) |
| EVM Version | cancun |

### Step 6: Click Verify

StoryScan will recompile the source and compare the deployed bytecode. If it matches, the contract shows a green checkmark.

### Step 7: Repeat for All Story Protocol Contracts

Verify these contracts on StoryScan (Chain 1514):

|---|---|---|
| 1 | StoryAttestationService | **1** |
| 2 | StoryOrchestrationService | 200 |
| 3 | StoryAttestationBridge | 200 |

## 4. ABI Bytecode Format Reference

### Where to Find the ABI and Bytecode

Each contract's ABI and bytecode is in the `artifacts/` directory:

```
artifacts/contracts/[ContractName].sol/[ContractName].json
```

### JSON Artifact Structure

```json
{
  "_format": "hh-sol-artifact-1",
  "contractName": "StoryAttestationService",
  "sourceName": "contracts/StoryAttestationService.sol",
  "abi": [
    {
      "inputs": [],
      "name": "functionName",
      "outputs": [{"type": "uint256"}],
      "stateMutability": "view",
      "type": "function"
    }
  ],
  "bytecode": "0x608060...",
  "deployedBytecode": "0x608060...",
  "linkReferences": {},
  "deployedLinkReferences": {}
}
```

### Key Fields for Verification

| Field | Purpose | Format |
|---|---|---|
| `abi` | Contract interface (functions, events) | JSON array |
| `bytecode` | Creation bytecode (used during deployment) | Hex string starting with `0x` |
| `deployedBytecode` | Runtime bytecode (on-chain) | Hex string starting with `0x` |

### Extract ABI for a Specific Contract

```bash
# Extract ABI only
cat artifacts/contracts/StoryAttestationService.sol/StoryAttestationService.json \
  | python3 -c "import sys,json; print(json.dumps(json.load(sys.stdin)['abi'], indent=2))" \
  > StoryAttestationService.abi.json

# Extract bytecode only
cat artifacts/contracts/StoryAttestationService.sol/StoryAttestationService.json \
  | python3 -c "import sys,json; print(json.load(sys.stdin)['bytecode'])" \
  > StoryAttestationService.bytecode.hex
```

## 5. JSON Artifacts Location & Format

### Repository Locations

| Artifact | Path |
|---|---|
| Compiled ABIs | `artifacts/contracts/[Name].sol/[Name].json` |
| Build Info (Standard JSON) | `artifacts/build-info/*.json` |
| Verification Hashes | `pinata-verification-hashes.json` |
| ABI Proof (full) | `abi-proof.json` |
| Deployment Config | `deployment-config.story.json` / `deployment-config.base.json` |
| Verification Artifacts | `verification-artifacts/[Name].storyscan.json` |

### Per-Contract Verification Artifact Format

Each file in `verification-artifacts/` follows this format:

```json
{
  "contractName": "StoryAttestationService",
  "chainId": 1514,
  "network": "Story Protocol Mainnet",
  "compiler": "v0.8.26+commit.8a97fa7a",
  "evmVersion": "cancun",
  "optimization": { "enabled": true, "runs": 1 },
  "viaIR": true,
  "license": "MIT",
  "constructorArguments": "0x",
  "abi": ["...full ABI array..."],
  "bytecodeKeccak256": "0x8df65dbd...",
  "deployedBytecodeKeccak256": "0xa3e5b1a8...",
  "abiSha256": "0x9dad5377...",
  "deployedBytecodeSize": 23342,
  "evmLimitCheck": "PASS"
}
```

## 6. Story Protocol Staking

### Option A: Stake via Story Explorer

1. Go to https://explorer.story.foundation/staking
2. Connect your wallet (MetaMask with Story Protocol network)
3. Select a validator (e.g., StakeMe: `story-peer.stakeme.pro:26656`)
4. Enter the amount of IP to stake
5. Click **Delegate** and confirm in MetaMask

### Option B: Stake via CLI

```bash
# Using story CLI
story staking delegate \
  --validator-addr <VALIDATOR_ADDRESS> \
  --amount 1000000000000000000 \
  --from 0x5EEFF17e12401b6A8391f5257758E07c157E1e45
```

## 7. Depositing Contracts into Wallets

### Add Token to MetaMask

After deployment, add the token contracts (AngelCoin, ResilienceToken) to MetaMask:

1. Open MetaMask
2. Switch to the correct network (Story or Base)
3. Click **Import Tokens**
4. Paste the deployed contract address
5. MetaMask auto-detects the token symbol and decimals
6. Click **Add Custom Token** > **Import**

### Add Token to Coinbase Wallet

1. Open Coinbase Wallet
2. Tap **Receive** > **More assets**
3. Paste the token contract address
4. The token appears in your balance

### Add Token to Safe Wallet

1. Open https://app.safe.global
2. Select Safe `0xd314BE0a27c73Cd057308aC4f3dd472c482acc09`
3. Go to **Assets** > **Custom tokens**
4. Paste the deployed contract address
5. Token balance appears automatically

## 8. Morpho Protocol Lending & Borrowing Setup

### Step 1: Authorize Safe on Morpho Blue

Using Safe Transaction Builder:

1. Open https://app.safe.global > Apps > **Transaction Builder**
2. **To Address:** `0xBBBBBbbBBb9cC5e90e3b3Af64bdAF62C37EEFFCb`
3. **ABI:** Auto-loads (verified on Basescan)
4. **Method:** `setAuthorization(address authorized, bool isAuthorized)`
5. **Parameters:**
   - authorized: `0xd314BE0a27c73Cd057308aC4f3dd472c482acc09`
   - isAuthorized: `true`
6. **Add Transaction** > **Create Batch** > **Send**
7. Collect 3-of-5 signatures from Safe owners
8. Execute

### Step 2: Supply Collateral

1. In the Morpho app (https://app.morpho.org), connect the Safe wallet
2. Select a market (BTC or ETH collateral)
3. Supply USDC as loan token
4. The IP collateral is referenced via the on-chain UCC-1 filing

### Step 3: Borrow Against IP

1. After supplying collateral, select **Borrow**
2. Choose the amount (within the LLTV of 86%)
3. Confirm the transaction (3-of-5 Safe approval)
